# ocr_utils

Утилита для постобработки PDF-сканов журналов: разбивка разворотов на отдельные страницы и добавление OCR-слоя максимального качества с сохранением исходных изображений.

## Ключевые особенности

- ✅ **Сохранение исходных изображений** — финальный PDF содержит оригинальные изображения без перекодирования и искажений
- ✅ **Качественный OCR** — двухэтапный процесс с применением deskew/clean/rotate для улучшения распознавания
- ✅ **Lossless JPEG crop** — разбивка разворотов без потери качества через `jpegtran`
- ✅ **Параллельная обработка** — многопоточная обработка директорий
- ✅ **Гибкая настройка** — управление параметрами OCR через CLI и API

## Алгоритм работы

### Шаг 1: Разбивка разворотов на страницы

1. Для каждой страницы исходного PDF определяем соотношение сторон (aspect ratio = ширина / высота).
2. Если aspect ratio ≥ **1.152** (`288/250`) — считаем страницу разворотом и разбиваем пополам по вертикали.
3. Если aspect ratio < 1.152 — это обычная страница, оставляем как есть.
4. **Копирование без перекодирования:**
   - Для полных страниц: прямое копирование через `insert_pdf()`
   - Для разворотов: копирование части страницы через `show_pdf_page()` с параметром `clip`
   - Это сохраняет исходные изображения без перекодирования и искажений
5. Результат — **промежуточный PDF #1** с разбитыми страницами и исходными изображениями.

### Шаг 2: Двухэтапный OCR для максимального качества

**Проблема:** Параметры OCR (deskew, clean, rotate) улучшают распознавание, но изменяют изображения.

**Решение:** Двухэтапный процесс:

#### Этап 2.1: OCR с обработкой (промежуточный PDF #2)
1. Промежуточный PDF #1 передаётся в `ocrmypdf` с параметрами:
   - **oversample** = 600 DPI (2× увеличение от исходных 300 DPI)
   - **deskew** = False (по умолчанию, можно включить)
   - **clean** = True (очистка шума)
   - **rotate-pages** = True (автоповорот по ориентации текста)
   - **language** = rus
   - **optimize** = 0 (не пережимать изображения)
   - **skip-text** = True (не перераспознавать страницы с текстом)
2. Результат — промежуточный PDF #2 с изменёнными изображениями, но **качественным OCR-слоем**.

#### Этап 2.2: Перенос текстового слоя
1. Извлекаются текстовые операторы из OCR XObject'ов промежуточного PDF #2.
2. Текстовые операторы встраиваются напрямую в страницы промежуточного PDF #1 (с исходными изображениями).
3. Копируются шрифты из промежуточного PDF #2.
4. Результат — **финальный PDF** с исходными изображениями + качественный OCR-слой.

## API

### `process_single_pdf` — обработка одного PDF

```python
from ocr_utils import process_single_pdf

process_single_pdf(
    src_pdf="input.pdf",              # Путь к исходному PDF
    dst_pdf="output.pdf",             # Путь к выходному PDF
    pages=None,                       # None=все, [0,2,5]=список, slice(1,10)=диапазон
    tmp_dir=None,                     # None=авто (tempfile), или путь к директории
    language="rus",                   # Язык OCR (по умолчанию: rus)
    oversample_dpi=600,               # DPI оверсемплинга (по умолчанию: 600)
    deskew=False,                     # Выравнивание наклона (по умолчанию: False)
    clean=True,                       # Очистка шума (по умолчанию: True)
    rotate_pages=True,                # Автоповорот (по умолчанию: True)
    intermediate_pdf_path=None,       # Путь для сохранения промежуточного PDF #2 (опционально)
)
```

**Параметры:**
- `src_pdf` — путь к исходному PDF
- `dst_pdf` — путь к выходному PDF
- `pages` — какие страницы обрабатывать:
  - `None` — все страницы
  - `[0, 2, 5]` — список индексов (0-based)
  - `slice(1, 10)` — диапазон страниц
- `tmp_dir` — директория для временных файлов (по умолчанию создаётся автоматически)
- `language` — язык(и) Tesseract для OCR
- `oversample_dpi` — DPI для оверсемплинга перед OCR (рекомендуется 600 для сканов 300 DPI)
- `deskew` — выравнивать ли наклон страниц (применяется только к промежуточному PDF #2)
- `clean` — очищать ли шум (применяется только к промежуточному PDF #2)
- `rotate_pages` — автоповорот страниц по ориентации текста (применяется только к промежуточному PDF #2)
- `intermediate_pdf_path` — путь для сохранения промежуточного PDF #2 после OCR с обработкой (для отладки)

### `process_directory` — рекурсивная обработка директории

```python
from ocr_utils import process_directory

results = process_directory(
    src_dir="scans/",                 # Исходная директория
    dst_dir="output/",                # Выходная директория (создаётся автоматически)
    workers=None,                     # None = 3/4 ядер, или конкретное число
    language="rus",                   # Язык OCR
    oversample_dpi=600,               # DPI оверсемплинга
    deskew=False,                     # Выравнивание наклона
    clean=True,                       # Очистка шума
    rotate_pages=True,                # Автоповорот
)
# results: dict {относительный_путь: None (успех) или строка ошибки}
```

**Параметры:**
- `src_dir` — исходная директория с PDF
- `dst_dir` — выходная директория (создаётся автоматически, структура сохраняется)
- `workers` — количество параллельных процессов:
  - `None` — автоматически (3/4 от количества ядер CPU)
  - Число — конкретное количество воркеров
- Остальные параметры — см. `process_single_pdf`

**Примечание:** Структура директорий сохраняется. Если файл был в `scans/1931/issue1.pdf`, результат будет в `output/1931/issue1.pdf`.

## CLI

```bash
# Один файл
ocr-utils single input.pdf output.pdf
ocr-utils single input.pdf output.pdf --pages "0,2,5"
ocr-utils single input.pdf output.pdf --pages "1:10"
ocr-utils single input.pdf output.pdf --language rus --dpi 600

# Директория
ocr-utils dir scans/ output/
ocr-utils dir scans/ output/ --workers 4
ocr-utils dir scans/ output/ --language rus --dpi 600

# Управление параметрами OCR
ocr-utils single input.pdf output.pdf --deskew        # включить выравнивание наклона
ocr-utils single input.pdf output.pdf --no-clean      # отключить очистку шума
ocr-utils single input.pdf output.pdf --no-rotate     # отключить автоповорот

# Комбинация параметров
ocr-utils single input.pdf output.pdf --dpi 900 --deskew --clean --rotate
ocr-utils dir scans/ output/ --workers 8 --dpi 600 --no-deskew
```

**Основные опции:**
- `--pages` — какие страницы обрабатывать (для `single`)
  - `"all"` — все страницы (по умолчанию)
  - `"0,2,5"` — список индексов (0-based)
  - `"1:10"` — диапазон страниц (Python slice)
- `--tmp-dir` — директория для временных файлов
- `--language` — язык OCR (по умолчанию: `rus`)
- `--dpi` — DPI оверсемплинга (по умолчанию: `600`)
- `--workers` — количество воркеров для `dir` (по умолчанию: 3/4 ядер)

**Флаги OCR (по умолчанию: deskew=OFF, clean=ON, rotate=ON):**
- `--deskew` / `--no-deskew` — выравнивание наклона страниц
- `--clean` / `--no-clean` — очистка шума
- `--rotate` / `--no-rotate` — автоповорот страниц

## Технические детали

### Двухэтапный OCR

Алгоритм разработан для решения противоречия: параметры OCR (deskew, clean, rotate) улучшают распознавание, но изменяют изображения.

**Решение:**
1. Промежуточный PDF #1 создаётся через `show_pdf_page()` с параметром `clip` — это копирует часть страницы без перекодирования изображений.
2. Промежуточный PDF #2 создаётся через `ocrmypdf` с обработкой — изображения изменяются, но OCR качественный.
3. Текстовые операторы извлекаются из OCR XObject'ов PDF #2 и встраиваются в PDF #1.
4. Финальный PDF содержит исходные изображения + качественный OCR-слой.

### Копирование без перекодирования

- **Полные страницы:** `insert_pdf()` — прямое копирование страницы
- **Развороты:** `show_pdf_page(clip=...)` — копирование части страницы без перекодирования
- **Результат:** Размер финального PDF близок к исходному (без учёта текстового слоя)

### Перенос текстового слоя

Используется `pikepdf` для работы с PDF на низком уровне:
1. Извлекаются текстовые потоки из OCR XObject'ов
2. Фильтруются ссылки на изображения (`/Im...Do`)
3. Текстовые операторы встраиваются напрямую в страницы
4. Копируются шрифты из промежуточного PDF

Это избегает ошибок "cannot find XObject resource" и сохраняет исходные изображения.
